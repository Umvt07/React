# React + Tailwind + Props Complete Notes

03. Tailwind CSS & Props in React - Detailed Notes
1. Tailwind CSS Configuration (Vite)
Why Tailwind?: It is a utility-first CSS framework that allows for rapid UI development without leaving your JSX file (04:00).
Installation Steps (05:10):
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p (This creates tailwind.config.js and postcss.config.js).
Configuration (06:35):
Update tailwind.config.js to include template paths so Tailwind can scan your files: content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"]
Add the following directives to your src/index.css file: @tailwind base; @tailwind components; @tailwind utilities;
Usage: In JSX, use the className attribute instead of the standard class attribute (08:50).
2. Understanding Props
Concept: Props (properties) make your components reusable. Instead of hardcoding values, you pass data from a parent component to a child component (14:07).
Component Structure (15:27):
It is best practice to keep components in a src/components/ directory.
Passing Props (18:25):
In App.jsx, pass custom attributes: ``.
Receiving Props (23:55):
The child component receives an object containing all passed properties: function Card(props) { ... }
Destructuring: To keep code cleaner, destructure the props directly in the function arguments (24:35): function Card({username, btnText}) { ... }
3. Advanced Props & Best Practices
Data Types: You can pass strings, variables, numbers, objects, and arrays by wrapping them in curly braces {} (21:00).
Default Values (28:40):
To ensure components don't break if a prop is missing, use default parameter syntax: function Card({username, btnText = "Visit Me"}) { ... }
Key Takeaway: React does not force strict segregation of HTML/CSS/JS. Instead, it groups them by component logic, making code easier to maintain and reuse across your application (14:30).
