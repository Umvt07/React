# React Fiber Notes

React Fiber is the new reconciliation engine introduced in React 16.

Earlier React used a simple recursive rendering algorithm where once rendering started React could not stop the work in between and it had to complete the entire rendering process in one go.

Because of this large UI updates could block the main thread and cause:
- laggy animations
- delayed user interactions
- bad scrolling performance

To solve this problem React introduced:
→ Fiber Architecture

--------------------------------------------------

# What is Fiber?

Fiber is a complete reimplementation of React's reconciliation algorithm.

Fiber allows React to:
- pause work
- resume work
- split work into small units
- prioritize important updates
- reuse previous work
- cancel unnecessary work

Because of this React applications become smoother and more responsive.

--------------------------------------------------

# Main Idea of Fiber

Earlier React rendering was:

→ synchronous

which means once rendering started it could not stop until entire component tree was rendered.

Fiber made rendering:

→ asynchronous and interruptible

This means React can now:
- pause rendering
- do higher priority work first
- continue rendering later

--------------------------------------------------

# Example

Suppose React is rendering a very large component tree and user suddenly clicks a button.

Old React:
- continues rendering entire tree first
- user interaction waits

Fiber:
- pauses rendering work
- handles user click immediately
- resumes rendering later

This improves user experience significantly.

--------------------------------------------------

# Fiber Tree

React creates a special Fiber Tree internally.

Each component becomes a Fiber node.

A Fiber node stores:
- component type
- props
- state
- parent reference
- child reference
- sibling reference
- update information

This tree helps React efficiently manage rendering work.

--------------------------------------------------

# Incremental Rendering

Fiber breaks rendering work into small chunks.

This is called:
→ Incremental Rendering

Instead of blocking browser for long time React performs rendering in smaller units.

This keeps UI responsive.

--------------------------------------------------

# Priorities in Fiber

Fiber can assign different priorities to updates.

Examples:
- button click → high priority
- animation → high priority
- background data loading → low priority

This helps React perform important updates first.

--------------------------------------------------

# Double Buffering

Fiber keeps:
- current tree
- work-in-progress tree

React prepares updates in work-in-progress tree first.

After completion React swaps trees efficiently.

--------------------------------------------------

# Phases in Fiber

There are mainly two phases.

1. Render Phase
- creates Fiber tree
- calculates changes
- can pause/resume/interupt

2. Commit Phase
- applies changes to real DOM
- updates UI
- cannot be interrupted

--------------------------------------------------

# Fiber and Reconciliation

Fiber is the engine that powers reconciliation.

Reconciliation means:
→ comparing old virtual DOM with new virtual DOM to find changes.

Fiber improves reconciliation performance significantly.

--------------------------------------------------

# Why Fiber is Important?

Fiber enables modern React features like:
- Concurrent Rendering
- Suspense
- Transitions
- Better animations
- Smooth UI updates

--------------------------------------------------

# Simple Definition

Fiber is React's modern reconciliation engine which allows React to perform rendering work incrementally, asynchronously, and with priorities to improve performance and user experience.

--------------------------------------------------

# Flow

State Update
      ↓
New Fiber Tree Created
      ↓
Diffing/Reconciliation
      ↓
Priority Scheduling
      ↓
Render Phase
      ↓
Commit Phase
      ↓
